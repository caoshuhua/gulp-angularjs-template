1、配置说明: (详细属性请参考http://issues.wenzhixin.net.cn/bootstrap-table/,源码详见https://github.com/wenzhixin/bootstrap-table)

  1. classes: 默认值'table table-hover':

    该属性指定的样式类会被加入table dom class上

    <table class="table table-hover" style="width: 1121px;">
.....
</table>
  2. locale: 国际化:

    当引入bootstrap-table-zh-CN.js时，会自动设置该参数, 例如bootstrap-table-zh-CN.js最后一行代码: 

     $.extend($.fn.bootstrapTable.defaults, $.fn.bootstrapTable.locales['zh-CN']);
  3. height: [Number]设置表格的高度

    其实就是设置div..fixed-table-container的高度，计算方法为: height = this.options.height - toolbarHeight - paginationHeight。 bootstrap-table 对应的dom结构如下

    <div class="bootstrap-table">
	<div class="fixed-table-toolbar"></div>
	<div class="fixed-table-container"></div>
	<div class="fixed-table-header"></div>
	<div class="fixed-table-body"></div>
	<div class="fixed-table-footer"></div>
	<div class="fixed-table-pagination"></div>
</div>
  4. undefinedText: 默认值为'-':

    当 单元格的值满足(typeof value === 'undefined' || value === null)表达式为true时，对应的单元格会用-代替

  5. showPaginationInfo: [boolean],默认为false: 是否显示分页信息
  6. sortName: 默认值为undefined:

    指定起始排序字段，当点击sortable 为true的列时，该值会被覆盖为当前列的值

  7. sortOrder: 默认值为asc:

    指定起始排序类型，当点击sortable 为true的列时，该值会被覆盖为当前列的值

  8. striped: 默认值为false:

    是否启用换行色

  9. columns: 默认值为[[]]:

    可以通过代码来配置列，也可以通过html的方式配置列信息, 如果两者同时存在时,columns配置优先, 注意两者不是覆盖，而是属性替换

  10. data: 表格数据:

    *

  11. dataField: 结果集数据字段,默认为rows:

    例如服务端返回的数据结果:{rows: [], total: 100}

  12. method: 默认为 get:

    *

  13. url: 默认undefined

    *

  14. ajax: 默认undefined

    指定自定义ajax服务对象

  15. cache: 默认true

    是否启用jQuery.ajax缓存

  16. contentType: 默认 'application/json'

    设定ajax发送的数据类型

  17. dataType: 默认 'json'

    设置服务端返回的数据类型

  18. ajaxOptions: 默认 {}

    设置ajax参数

  19. pagination: 默认 false

    是否启用扉页

  20. onlyInfoPagination: 默认false,

    是否仅显示分页信息

  21. sidePagination: 默认client,

    设置分页模式： 服务端 还是server 客户端client

  22. 这些属性都是应该设置分页参数的

扩展插件配置

  1. bs-table-control: 表格控件指令, 该指令的作用是生成bootstrap-table表格控件,有关控件设计描述如下 

    /**
 * 优先级为100, 在angular中 优先级低的指令会被优先执行
 * @type {Number}
 */
priority: 100,
/**
 * bs-table-control指令是依赖于ngModel指令的, 所以ngModel指令必须被设置
 * @type {Array}
 */
require: ['ngModel'],
/**
 *bs-table-control能够从父scope中绑定以下属性:
 bsTableControl: 表格控件的配置对象
 avdQueryParams: 表格控件的高级查询参数
 tableData: 表格控件数据绑定对象
 onClickRow: 行点击事件,参数类型{row: ?[Object]}
 onDblClickRow: 行双击事件,参数类型{row: ?[Object]}
 onClickCell: 列点击事件,参数类型{row: ?[Object], row: ?[String], value: ?}
 onClickCell: 列双击事件,参数类型{row: ?[Object], row: ?[String], value: ?}
 onCellCommand: 单元格Command事件,参数类型{?,?},
 onServerSearch: 查询事件{query: ?[Object]}
 * @type {Object}
 */
scope: {
    bsTableControl: '=',
    avdQueryParams: '=',
    tableData: '=ngModel',
    onClickRow: '&',
    onDblClickRow: '&',
    onClickCell: '&',
    onDblClickCell: '&',
    onCellCommand: '&',
    onServerSearch: '&',
}
/**
 * 自动将table.$el对象注入到$Scope中, 这样我们就可以在controller自由的调用table方法
 * @type {[type]}
 */
table-reference: ?[string]

  2. 高级查询插件: 高级查询插件，启用该插件时，表格上方会显示 按钮，点击该按钮会弹出对话框, 并且高级查询表单是支持angular template的,具有非常好的灵活性: 

    /**
 * 该模板必须被$templateCache缓存管理，否则会找不到该模板, 如何将模板加入到$templateCache缓存中呢，方式有两种：
 1.在页面中通过以下标签定义模板，会被该缓存自动管理:

    <script type="text/ng-template" id="table-detail-templateId">
  	 模板内容
</script>

    
 2.手动添加到$templateCache中,例如$templateCache.put('templateId', templateContent), 建议定义一个angular template config service(angular模板配置服务类)，专为用于管理平台的所有模板数据
 * @type {[type]}
 */
avdSearchTpl: ?['angular template id'],
/**
 * 是否启用高级查询插件
 * @type {[type]}
 */
advancedSearch: ?[boolean],
/**
 * 绑定高级查询参数
 * @type {Array}
 */
avd-query-params-bind: [$scope.modal]

  3. 分组插件: 该组件用于表格数据的分组、汇总，目前支持多列分组、Sum聚合统计 

    /**
 * 是否启用分组
 * @type {Array}
 */
groupBy: [boolean],
/**
 * 是否启用分组sum汇总计算，当状态为启用时，他会对所有groupBySumGroup=true的列类字段进行汇总统计，并显示在对应字段的分组列中, 未来会根据需求扩展出一些个性化的功能
 */
groupBySumGroup[boolean],
/**
 * 分组字段，在table标签中的配置是:data-group-by-field="[field1, field2]"
 */
groupByField: ? [array[(string)]]

  4. 详细插件: 该插件是用来显示每条record数据扩展信息的,也就是我们常说的detailView, 该插件已经集成了angular，这样我们可以更加灵活的定制出个性化、复杂的detailView信息页面, 默认会向scope中传递$tab_row_Record 和 $tab_row_index两个modal参数, 如果detailView内部会创建新的angular scope的话，我们通过scope.$parent获取该参数: 

    /**
 * 是否启用detail插件
 */
detailView: [boolean],
/**
 *  detailView angular模板,该模板必须存放在$templateCache中
 * @type {Array}
 */
detailTpl: [angular template]

  7. Cell Command: 支持通过formatter定义单元格的操作指令, 例如:

    /**
 * 指令处理函数配置属性为onCellCommand, 参数信息来自于cell-command节点的data-*属性

 * @param  {[type]} index [description]
 * @param  {[type]} row   [description]
 * @return {[type]}       [description]
 */
formatter: function(index, row){
	return '<a href="#" class="cell-command" data-record-id="'+ row.id +'"><i class="fa fa-line-chart"></i></a>'
}
