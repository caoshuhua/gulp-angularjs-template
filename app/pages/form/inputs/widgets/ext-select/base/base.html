Select Base

Select Base 控件是一个基础控件，真正的选择控件是由 Select Base 和 Picker构成, 接下来让我们看一看Select Base是什么

html 

<form name="exampleForm" class="form-inline">
	<div class="input-group">
	<input type="text"
				ng-model="selects"
				ui-select-base
				data-picker-width="400",
				data-picker-height="100",
				data-trigger-selector=":parent:button"
				class="form-control"
				/>
				<span class="input-group-btn">
            		<button type="button" class="btn btn-default" >
            		<i class="glyphicon glyphicon-chevron-down"></i>
            		</button>
          		</span>
		</div>
</form>

配置说明

说明: 框架中所有控件的可配属性格式:
1: 如果在节点中设置，必须是data-*的格式, 例如(data-template-tpl)
2:如果在controller中设置 就不要加data前缀了,例如(templateTpl)

  1. ui-select-base: Select Base指令 通过该指令设置表单参数:

    格式: { options: { //表单配置参数 }, datas: [] //数据 }

  2. ng-model: Select Base指令依赖于ng-model指令, 用户绑定表单值,所以ng-model是必须的
  3. ng-disabled: 表单禁用控制指令
  4. pickerWidth: 设置下来面板的宽度
  5. pickerHeight: 设置下来面板的高度
  6. triggerSelector:[string],默认':self', 设置下拉面板触发元素, 语法格式:

    :self 当前input元素
    :parent:selector 代表 $(currentEl).parent().find(selector).eq(0)
    selector 表示$(select)

  7. singleSelect:[boolean] 默认false, 是否单选
  8. textField:[string],默认'text', 文本域字段
  9. idField:[string],默认'id' 业务域字段
  10. templateTpl:[string] picker模板
  11. plain:[boolean],默认false, true代表templateTpl为模板内容, false代表为templateTpl为模板Id
  12. containParentNode: [boolean] 默认false: 设置树选择时，是否包含父节点
